introduction

CSS (Cascading Style Sheets) is a stylesheet language used to describe the presentation of a document written in HTML or XML. CSS defines how HTML elements should be displayed on screen, paper, or in other media. It allows web developers to separate content (HTML) from design (CSS), enabling more flexibility and control in the specification of presentation characteristics

Key Concepts of CSS

1.Selectors

These are used to target HTML elements that you want to style. Common selectors include element selectors, class selectors, and ID selectors

2.Properties

These define the aspects of the HTML elements you want to change (e.g., color, font-size, margin)

3.Values

These are the settings you apply to properties (e.g., red, 16px, 20px)

colors

The rgba function is similar to rgb, but includes an alpha channel for opacity

The hsl function defines colors using the Hue, Saturation, and Lightness model

The hsla function adds an alpha channel to hsl colors

CSS custom properties allow you to define reusable color variables

The currentColor keyword applies the current color of an element to its children or other properties

The transparent keyword is used to make an element completely transparent

text in CSS

CSS provides a variety of properties to style text on a web page. These properties allow you to control the font, size, spacing, alignment, decoration, and other aspects of the text. Below is an explanation of the main CSS text properties with examples

1. color

Sets the color of the text

p {
color: blue;
}
2. font-family

Specifies the font of the text. You can list multiple fonts as fallbacks

body {
font-family: Arial, Helvetica, sans-serif;
}
3. font-size

Sets the size of the text

h1 {
font-size: 2em;
}
4. font-weight

Specifies the weight (or boldness) of the text

strong {
font-weight: bold;
}
.light-text {
font-weight: 300;
}
5. font-style

Specifies the style of the text (e.g., normal, italic, oblique)

em {
font-style: italic;
}
6. text-align

Specifies the horizontal alignment of the text

h1 {
text-align: center;
}
p {
text-align: justify;
}
7. text-decoration

Adds decoration to the text (e.g., underline, overline, line-through, none)

a {
text-decoration: none;
}
del {
text-decoration: line-through;
}
8. text-transform

Controls the capitalization of the text

h2 {
text-transform: uppercase;
}
.address {
text-transform: capitalize;
}
9. letter-spacing

Adjusts the space between characters

h1 {
letter-spacing: 2px;
}
10. line-height

Sets the height of the lines of text

p {
line-height: 1.5;
}
11. word-spacing

Adjusts the space between words

p {
word-spacing: 5px;
}
12. text-indent

Indents the first line of the text block

p {
text-indent: 50px;
}
13. white-space

Controls how whitespace inside an element is handled

pre {
white-space: pre; /* Preserves whitespace and line breaks */
}

    position in CSS

    CSS offers several ways to position elements on a webpage. Here are the different position values available in CSS along with explanations and example code for each

    1.Static Positioning (default)

    This is the default positioning for all elements. Elements are positioned according to the normal document flow

    example 2.Relative Positioning

    Elements are positioned relative to their normal position. Using top, right, bottom, and left will move the element from its original position, but it will still occupy its original space

    example

    Relative positioning allows you to move an element relative to its normal position in the document flow. The element remains in the document flow, but its visual position can be adjusted with the top, right, bottom, and left properties

    example 3.Absolute Positioning

    Elements are positioned relative to their nearest positioned ancestor (i.e., the nearest ancestor with a position value other than static). If no such ancestor exists, it will be positioned relative to the initial containing block

    example 4.Fixed Positioning

    Elements are positioned relative to the browser window. They do not move when the window is scrolled

    example 5.Sticky Positioning

    Elements are positioned based on the user's scroll position. A sticky element toggles between relative and fixed positioning, depending on the scroll position. It is positioned relative until it crosses a specified threshold, then it is fixed

    media query in css

    A useful technique for applying styles based on the properties of the device or viewport displaying the content is CSS media queries. They make responsive design possible, so you can make websites that change to fit various screen sizes and gadgets. Below is a breakdown of different media inquiries along with samples of each.

    1. Basic Media Query

    A media query that applies styles based on the width of the viewport

    /* Styles for screens wider than 600px */
    @media (min-width: 600px) {
    body {
    background-color: lightblue;
    }
    }
    /* Styles for screens narrower than 600px */
    @media (max-width: 599px) {
    body {
    background-color: lightcoral;
    }
    }
    2. Combining Conditions with Logical Operators

    You can use and, or, and not to combine multiple conditions

    /* Styles for screens between 600px and 1200px */
    @media (min-width: 600px) and (max-width: 1200px) {
    body {
    background-color: lightgreen;
    }
    }
    /* Styles for screens wider than 1200px or in landscape orientation */
    @media (min-width: 1200px), (orientation: landscape) {
    body {
    background-color: lightyellow;
    }
    }
    /* Styles for screens that are not in portrait orientation */
    @media not all and (orientation: portrait) {
    body {
    background-color: lightgray;
    }
    }
    3. Targeting Device Characteristics

    Media queries can target various device characteristics such as resolution, aspect ratio, and color

    /* Styles for high-resolution screens (e.g., Retina displays) */
    @media (min-resolution: 2dppx) {
    body {
    background-color: lightpink;
    } }
    /* Styles for screens with an aspect ratio of 16:9 */
    @media (aspect-ratio: 16/9) {
    body {
    background-color: lightblue;
    }
    }
    /* Styles for screens that can display at least 8-bit color */
    @media (min-color: 8) {
    body {
    background-color: lightgreen;
    }
    }
    4. Orientation

    Target styles based on the orientation of the device (portrait or landscape)

    /* Styles for portrait orientation */
    @media (orientation: portrait) {
    body {
    background-color: lightblue;
    }
    }
    /* Styles for landscape orientation */
    @media (orientation: landscape) {
    body {
    background-color: lightcoral;
    }
    }
    5. Prefers Color Scheme

    Apply styles based on the user’s preferred color scheme (e.g., dark mode or light mode)

    /* Styles for users who prefer a dark color scheme */
    @media (prefers-color-scheme: dark) {
    body {
    background-color: black;
    color: white;
    }
    }
    /* Styles for users who prefer a light color scheme */
    @media (prefers-color-scheme: light) {
    body {
    background-color: white;
    color: black;
    }
    }
    6. Prefers Reduced Motion

    Adjust styles for users who prefer reduced motion for accessibility reasons

    /* Styles for users who prefer reduced motion */
    @media (prefers-reduced-motion: reduce) {
    * {
    animation: none;
    transition: none;
    }

    }

    CSS units

    Absolutely, let's go into more detail on each CSS unit: em, rem, vh, vw, and in, with both explanations and examples that cover their usage in HTML and CSS


    1.em
    Explanation
    * em is a relative unit
    * It is relative to the font-size of the element it is used on. If used for the font-size property, it is relative to the font-size of the parent element
    * em units are most commonly used for font sizes, margins, paddings, and spacing
    Example 2.rem
    Explanation
    * rem stands for "root em"
    * It is relative to the font-size of the root element (<html>)
    * This unit ensures consistency across different elements since they all refer to the same root font size
    Example 3.vh
    Explanation
    * vh stands for "viewport height"
    * 1vh is equal to 1% of the height of the viewport (the visible area of the browser window)
    * Useful for making elements responsive to the height of the viewport
    Example 4.vw
    Explanation
    * vw stands for "viewport width"
    * 1vw is equal to 1% of the width of the viewport
    * Useful for making elements responsive to the width of the viewport
    Example 5.in
    Explanation
    * in stands for inches
    * It is an absolute unit of measurement
    * Primarily used in print styles or for specific sizing requirements

    CSS animations

    CSS animations allow you to create complex animations with keyframes, specifying how an element should change at specific points in time. Unlike transitions, which are limited to simple property changes triggered by events like hover, animations provide more control and flexibility

    Key Concepts of CSS Animations

    1.@keyframes Rule
    Defines the intermediate steps in an animation sequence
    2.animation-name
    Specifies the name of the @keyframes animation
    3.animation-duration
    Defines how long the animation should take to complete one cycle
    4.animation-timing-function
    Specifies the speed curve of the animation
    5.animation-delay
    Defines when the animation will start
    6.animation-iteration-count
    Specifies the number of times the animation should play
    7.animation-direction
    Defines whether the animation should play in reverse on alternate cycles
    Example Code @keyframes Rule

    0%: Initial state with no transformation, a scale of 1, original background color, and a border-radius of 10px

    50%: The box moves 100px to the right, scales up to 1.5, changes the background color to red, and the border-radius to 50px
    75%: The box moves back to 50px to the right, scales down to 1.2, changes the background color to blue, and the border-radius to 25px
    100%: The box returns to its original position, scale, background color, and border-radius.

    z-index in css

    The z-index property in CSS controls the vertical stacking order of elements that overlap. Elements with a higher z-index value will appear in front of elements with a lower z-index value

    Syntax
    element {
    position: relative; /* or absolute, fixed, sticky */
    z-index: value; /* integer value */
    }
